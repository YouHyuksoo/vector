# VRL & 매핑

## 개요

설비 로그 파싱 코드와 Oracle 대상 매핑을 한 화면에서 관리합니다. 먼저 왼쪽에서 설비 유형을 선택합니다.

## VRL 탭

1. 샘플 로그를 입력합니다.
2. VRL 코드를 직접 작성하거나 AI 생성을 사용합니다.
3. **시뮬레이션**으로 파싱 결과와 오류를 확인합니다.
4. 검증된 코드를 TOML에 반영합니다.
5. 반영 후 안내에 따라 Vector Aggregator를 다시 불러옵니다.

## 매핑 탭

- 대상 유형을 **TABLE** 또는 **PROCEDURE**로 선택합니다.
- VRL의 `data.*` 필드를 Oracle 컬럼 또는 프로시저 파라미터에 연결합니다.
- 대상이 없으면 자동 생성 기능을 사용할 수 있습니다.
- 저장 전 필수 컬럼과 타입이 맞는지 확인합니다.

> 운영 로그에 즉시 영향을 줄 수 있으므로 VRL 적용 전 시뮬레이션을 반드시 통과시키세요.
