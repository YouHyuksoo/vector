# 시스템 설정

## 개요

서버, 데이터베이스, Redis, 큐 등 시스템 전반의 환경설정을 관리하는 페이지입니다.
하단에는 AI 모델 설정 섹션이 있어 VRL 시뮬레이터에서 사용할 AI를 구성할 수 있습니다.

## 전제조건

- Fastify 서버가 실행 중이어야 합니다
- 일부 설정 변경 후에는 서버 재시작이 필요합니다

## 사용 방법

### 설정 보기

페이지 진입 시 현재 시스템 설정이 6개 섹션으로 표시됩니다.

### 설정 편집

1. **편집** 버튼을 클릭하여 편집 모드에 진입합니다
2. 각 항목의 값을 수정합니다
3. **저장** 버튼을 클릭하여 저장합니다
4. **취소** 버튼을 클릭하면 변경사항이 되돌려집니다

### 설정 섹션

#### 서버 (Server)

| 항목 | 설명 | 예시 |
|------|------|------|
| HOST | 서버 바인드 IP | `0.0.0.0` |
| PORT | HTTP 포트 | `3100` |
| NODE_ENV | 실행 환경 | `production` |

> 서버 포트 변경 시 서버 재시작이 필요합니다.

#### Oracle

| 항목 | 설명 | 예시 |
|------|------|------|
| HOST | DB 서버 IP | `192.168.1.100` |
| PORT | 리스너 포트 | `1521` |
| SID | 데이터베이스 SID | `ORCL` |
| USER | 접속 계정 | `log_user` |
| PASSWORD | 비밀번호 | `****` |
| POOL_MIN | 최소 커넥션 풀 | `2` |
| POOL_MAX | 최대 커넥션 풀 | `10` |

> Oracle 연결 정보 변경 시 서버 재시작이 필요합니다.

#### Redis

| 항목 | 설명 | 예시 |
|------|------|------|
| HOST | Redis 서버 IP | `127.0.0.1` |
| PORT | Redis 포트 | `6379` |
| PASSWORD | 인증 비밀번호 | (비어있을 수 있음) |

#### 큐 (Queue)

| 항목 | 설명 | 기본값 |
|------|------|--------|
| CONCURRENCY | 동시 처리 수 | `5` |
| BATCH_SIZE | 배치 크기 | `10` |
| BATCH_TIMEOUT_MS | 배치 타임아웃 (ms) | `5000` |

#### 저장소 (Storage)

| 항목 | 설명 | 예시 |
|------|------|------|
| RAW_LOG_BASE_PATH | 원본 로그 저장 경로 | `/data/raw` |

#### 하트비트 (Heartbeat)

| 항목 | 설명 | 기본값 |
|------|------|--------|
| TTL_SECONDS | Agent 하트비트 만료 시간 (초) | `60` |

> 이 시간 내에 하트비트가 없으면 Agent가 오프라인으로 표시됩니다.

## AI 모델 설정

화면 하단에서 VRL 시뮬레이터의 AI 코드 생성에 사용할 모델을 구성합니다.

### 지원 모델

| 제공사 | 모델 |
|--------|------|
| **Google Gemini** | 2.5 Flash, 2.5 Pro, 2.0 Flash, 2.0 Flash-Lite |
| **Mistral AI** | Large, Small, Codestral, Nemo |
| **Anthropic Claude** | Sonnet 4, Haiku 4.5, Opus 4 |

### 설정 방법

1. 사용할 AI 제공사의 **API 키**를 입력합니다
2. 사용할 **모델**을 선택합니다
3. **활성화** 토글을 켭니다
4. **테스트** 버튼으로 API 연결을 확인합니다
   - 성공: 응답 시간(ms)이 표시됩니다
   - 실패: 오류 메시지가 표시됩니다

> API 키가 입력되어 있어야 활성화 토글이 작동합니다.
> 활성화된 모델은 VRL 시뮬레이터의 AI 생성 기능에서 사용할 수 있습니다.
