# 대시보드

## 개요

시스템 전체 상태를 한눈에 파악할 수 있는 메인 모니터링 화면입니다.
5초마다 자동으로 데이터가 갱신되어 실시간으로 인프라 상태를 확인할 수 있습니다.

## 전제조건

- Fastify 서버(포트 3100)가 실행 중이어야 합니다
- Redis, Oracle DB가 연결되어 있어야 정상적인 상태 정보가 표시됩니다

## 화면 구성

### 인프라 현황 카드

시스템의 핵심 컴포넌트 4가지의 상태를 표시합니다.

| 항목 | 표시 내용 | 정상 상태 |
|------|-----------|-----------|
| **Server** | Fastify 서버 운영 시간 | 녹색 "가동" |
| **Redis** | 연결 상태 | 녹색 "연결됨" |
| **Oracle** | 등록된 테이블 개수 | 녹색 "N 테이블" |
| **Vector** | 수집 엔진 실행 상태 | 녹색 "정상" |

- Vector 항목에는 **시작/정지** 버튼이 있어 수집 엔진을 직접 제어할 수 있습니다

### 서버 리소스 모니터링

인프라 현황 카드 하단에 서버의 하드웨어 자원 사용량이 프로그레스 바로 표시됩니다.

| 항목 | 표시 내용 | 경고 조건 |
|------|-----------|-----------|
| **CPU** | 사용률 %, 코어 수 | 90% 이상 빨간색 |
| **메모리** | 사용량 / 전체 (GB) | 90% 이상 경고 메시지 |
| **디스크** | 사용량 / 전체 (GB) | 90% 이상 경고 메시지 |

- 70% 미만: **녹색** (정상)
- 70% ~ 90%: **노란색** (주의)
- 90% 이상: **빨간색** + 깜빡임 경고 메시지 표시
- 5초마다 자동 갱신됩니다

### 큐 현황

BullMQ 작업 큐의 처리 상태를 보여줍니다.

- **대기**: 아직 처리되지 않은 로그 건수
- **처리중**: 현재 Oracle DB에 적재 중인 건수
- **완료**: 성공적으로 처리된 건수
- **실패**: 적재 실패한 건수 (오류 현황 페이지에서 상세 확인)

### 등록된 테이블

Oracle DB에 생성된 로그 저장 테이블 목록과 각 테이블의 컬럼 수를 표시합니다.

### 장비 수집기

각 설비 PC에 설치된 Agent의 온라인/오프라인 상태를 보여줍니다.

- **녹색 점**: 온라인 (하트비트 수신 중)
- **회색 점**: 오프라인 (하트비트 미수신)
- 각 카드에 설비 유형, 라인 코드, 설비 ID, 마지막 통신 시간이 표시됩니다
- 상단에 전체/온라인/오프라인 설비 수 통계가 있습니다
