# 수신기 설정

## 개요

서버에서 실행되는 Aggregator(수집 서버)의 TOML 설정을 편집하는 페이지입니다.
Aggregator는 모든 설비 Agent에서 보내는 로그를 수신하고, VRL로 파싱한 후, API 서버로 전달합니다.

## 전제조건

- Fastify 서버가 실행 중이어야 합니다
- `config/aggregator.toml` 파일이 서버에 존재해야 합니다

## 화면 구성

### 좌측: 설정 편집

#### 기본 설정

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 데이터 디렉토리 | Vector 내부 데이터 저장 경로 | `/data/vector` |
| Vector API IP | Vector 관리 API 바인드 IP | `0.0.0.0` |
| API 포트 | Vector 관리 API 포트 | `8686` |

#### Agent 수신

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 수신 IP | Agent 접속을 받을 IP | `0.0.0.0` |
| 수신 포트 | Agent 접속 포트 | `9000` |

> 이 포트가 방화벽에서 열려 있어야 Agent가 접속할 수 있습니다.

#### Raw 파일 저장

수집된 원본 로그를 파일로 저장하는 설정입니다.

| 항목 | 설명 |
|------|------|
| 저장 경로 패턴 | 설비 유형별 자동 분류 저장 (예: `/data/raw/{{ equipment_type }}/...`) |
| 버퍼 크기 | 파일 쓰기 버퍼 (MB 단위) |

#### 타겟 라우팅

설비 유형별로 데이터 저장 방식을 선택합니다.

- **테이블 INSERT**: Oracle 테이블에 직접 INSERT (기본)
- **프로시져 CALL**: Oracle 프로시져를 호출하여 저장

각 설비 유형별로 독립적으로 설정할 수 있습니다.
타겟 이름을 지정하지 않으면 `LOG_{설비유형}` 형식이 기본값으로 사용됩니다.

#### API 서버 전송

| 항목 | 설명 | 기본값 |
|------|------|--------|
| API URI | Fastify 서버 엔드포인트 | `http://localhost:3100/api/logs/ingest` |
| 배치 크기 | 한 번에 전송할 이벤트 수 | `10건` |
| 배치 주기 | 배치 전송 타임아웃 | `5초` |
| 버퍼 크기 | 전송 버퍼 (MB) | - |
| 초기 대기 | 재시도 시 첫 대기 시간 | `1초` |
| 최대 대기 | 재시도 최대 대기 시간 | `10초` |

#### TOML 직접 편집

**TOML 직접 편집** 토글을 열면 설정 파일의 전체 내용을 텍스트로 직접 편집할 수 있습니다.

> 직접 편집 시 TOML 문법 오류에 주의하세요.

### 우측: 변경 이력

설정 파일이 변경될 때마다 자동으로 백업이 생성됩니다.

#### 백업 목록 확인

각 백업 항목에는 다음 정보가 표시됩니다:
- **일시**: 백업 생성 시간
- **출처**: 변경 원인 (editor, vrl-apply, restore)
- **크기**: 파일 크기

#### 미리보기

**미리보기** 버튼을 클릭하면 해당 시점의 설정 파일 내용을 확인할 수 있습니다.

#### 설정 복구

1. 복구할 백업의 **복구** 버튼을 클릭합니다
2. 확인 팝업에서 **확인**을 클릭합니다
3. 현재 설정이 선택한 백업으로 교체됩니다 (교체 전 현재 설정도 자동 백업됩니다)

### 저장 및 적용

1. 설정 수정 후 **저장** 버튼을 클릭합니다
2. 저장 완료 후 "Vector 재시작" 모달이 표시됩니다
3. **재시작**: 변경된 설정을 즉시 적용 (Vector 엔진 재시작)
4. **나중에**: 설정만 저장하고 재시작은 나중에 수동으로 진행
