# Redis

## Redis가 뭔가요?

Redis는 **메모리 기반 초고속 저장소**입니다.
하드디스크 대신 RAM에 데이터를 저장해서 읽기/쓰기가 매우 빠릅니다.

이 시스템에서는 Oracle처럼 영구 데이터를 보관하는 용도가 아니라,
**임시로 빠르게 처리해야 하는 2가지 작업**에 사용합니다.

## 역할 1: 작업 큐 (BullMQ)

### 왜 큐가 필요한가?

로그가 한꺼번에 수천 건 들어오면 Oracle에 바로 넣으면 과부하로 터질 수 있습니다.
그래서 Redis에 **대기열(큐)**을 만들어 순서대로 처리합니다.

```
로그 대량 수신
    ↓
Redis 큐에 쌓기 (즉시 응답 — "접수 완료")
    ↓
워커가 동시 5개씩 꺼내서
    ↓
Oracle에 저장
```

### 큐 상태

대시보드의 **큐 현황** 카드가 바로 이 Redis 큐의 상태입니다.

| 상태 | 의미 |
|------|------|
| **대기** | 큐에 쌓여서 아직 처리 안 된 건수 |
| **처리중** | 지금 워커가 Oracle에 넣고 있는 건수 |
| **완료** | 성공적으로 Oracle에 저장된 건수 |
| **실패** | 3회 재시도에도 저장 실패한 건수 |

### 우선순위

- **ALARM 로그** → 우선순위 1 (최우선 처리)
- **일반 로그** → 우선순위 5

### 재시도

저장 실패 시 자동으로 3회 재시도합니다 (1초 → 2초 → 4초 간격).
3회 모두 실패하면 **실패** 상태로 보관되고, **오류 현황** 페이지에서 확인할 수 있습니다.

## 역할 2: 설비 하트비트 (온라인/오프라인 판정)

### 동작 원리

```
설비 Agent → "나 살아있어" (하트비트) → Redis에 60초 타이머로 저장
                                         ↓
                              60초 안에 다시 오면 → 타이머 리셋 (온라인 유지)
                              60초 안에 안 오면 → 자동 삭제 = 오프라인 판정
```

### 왜 Redis인가?

- 하트비트는 수십 대 설비에서 **매 몇 초마다** 들어옵니다
- 이걸 Oracle에 저장하면 너무 느립니다
- Redis는 메모리에서 처리하므로 **밀리초 단위**로 응답 가능
- TTL(Time To Live) 기능으로 만료 시 자동 삭제 — 별도 삭제 코드가 필요 없음

### 대시보드 연동

대시보드의 **장비 수집기** 카드가 이 Redis 데이터를 읽는 것입니다.

| 표시 | 의미 |
|------|------|
| 🟢 녹색 점 | Redis에 하트비트가 존재 → 온라인 |
| ⚫ 회색 점 | Redis에서 TTL 만료로 삭제됨 → 오프라인 |

## Oracle vs Redis 비교

| | Oracle | Redis |
|---|--------|-------|
| **저장 위치** | 하드디스크 | RAM (메모리) |
| **속도** | 상대적으로 느림 | 초고속 (밀리초) |
| **데이터 수명** | 영구 보관 | 임시 (TTL 만료 시 삭제) |
| **용도** | 로그 데이터, 매핑 설정 | 작업 큐, 실시간 상태 |
| **서버 재시작 시** | 데이터 유지 | 데이터 사라짐 (큐는 재시작 후 복구 가능) |

## 설정

**설정** 페이지에서 Redis 연결 정보를 관리합니다.

| 항목 | 설명 | 기본값 |
|------|------|--------|
| HOST | Redis 서버 IP | `127.0.0.1` |
| PORT | Redis 포트 | `6379` |
| PASSWORD | 인증 비밀번호 | (없음) |

## 장애 시 영향

| 상황 | 영향 |
|------|------|
| Redis 중단 | 로그 큐 적재 불가, 하트비트 저장 불가, 대시보드 상태 표시 안 됨 |
| Redis 재시작 | 큐에 쌓여있던 미처리 로그 유실 가능, 하트비트 전체 리셋 (일시적 오프라인 표시) |
| Oracle 중단 | 큐에 로그가 쌓이기만 하고 적재 불가 (Redis가 버퍼 역할) |

> Redis가 중단되면 시스템 전체가 영향을 받으므로, 안정적으로 운영하는 것이 중요합니다.
