# 타겟 매핑

## 개요

설비에서 수집한 로그 데이터를 Oracle DB에 어떻게 저장할지 매핑을 설정하는 페이지입니다.
수집된 로그의 각 필드를 Oracle 테이블의 컬럼이나 프로시져의 파라미터에 연결합니다.

## 전제조건

- Oracle DB가 연결되어 있어야 합니다
- 매핑 대상 **테이블** 또는 **프로시져**가 Oracle에 미리 생성되어 있어야 합니다
- 로그 유형별 매핑을 사용하려면 **VRL 시뮬레이터**에서 파싱 룰이 먼저 설정되어야 합니다

## 사용 방법

### 1단계: 타겟 유형 선택

화면 상단에서 두 가지 모드 중 하나를 선택합니다:

- **테이블**: Oracle 테이블에 직접 INSERT
- **프로시져**: Oracle 프로시져/패키지를 CALL

### 2단계: 대상 선택 (좌측 패널)

**테이블 모드**:
- Oracle에 존재하는 테이블 목록이 표시됩니다
- 검색 필터로 원하는 테이블을 찾을 수 있습니다
- 테이블을 클릭하면 우측에 컬럼 목록이 표시됩니다

**프로시져 모드**:
- Oracle에 존재하는 프로시져/패키지 목록이 표시됩니다
- 패키지 아이콘(📦)으로 패키지와 일반 프로시져를 구분합니다
- 프로시져를 클릭하면 우측에 파라미터 목록이 표시됩니다

### 3단계: 설비 유형 선택

매핑에 사용할 설비 유형(로그 유형)을 선택합니다.

- **파싱 완료**(초록색): VRL 파싱이 설정된 설비 — 드롭다운에서 소스 필드 선택 가능
- **미파싱**(점선): 아직 VRL이 설정되지 않은 설비 — 선택 불가

> 설비 유형을 선택하지 않으면 소스 필드를 텍스트로 직접 입력해야 합니다.

### 4단계: 소스 필드 매핑

#### 테이블 모드

| 컬럼 | 설명 |
|------|------|
| 컬럼명 | Oracle 테이블의 컬럼 이름 |
| 타입 | 데이터 타입 (VARCHAR2, NUMBER 등) |
| Nullable | NULL 허용 여부 |
| 소스 필드 | 매핑할 로그 필드 선택 또는 입력 |
| 필수 | 적재 시 필수 여부 (Y/N) |

- **설비 유형 선택 시**: 드롭다운에서 파싱된 필드를 선택합니다
- **설비 유형 미선택 시**: 텍스트 필드에 직접 입력합니다 (예: `log.field_name`)
- 매핑된 컬럼은 좌측에 녹색 체크(✓)가 표시됩니다

#### 프로시져 모드

| 컬럼 | 설명 |
|------|------|
| 순서 | 파라미터 순서 |
| 파라미터명 | Oracle 프로시져 파라미터 이름 |
| 타입 | 데이터 타입 |
| 방향 | IN / OUT / IN OUT |
| 소스 필드 | 매핑할 로그 필드 |
| 필수 | 필수 여부 |

**호출 모드**:
- **NAMED (개별 파라미터)**: 파라미터명으로 호출 — 일반적인 프로시져 호출
- **ARRAY (배열 파라미터)**: 배열로 한번에 전달 — Oracle Collection 타입명 입력 필요

### 5단계: 자동 매핑 (선택사항)

**자동 매핑** 버튼을 클릭하면 컬럼명과 소스 필드명이 일치하는 항목을 자동으로 매핑합니다.

- 예: Oracle 컬럼 `INSPECTOR` → 소스 필드 `data.INSPECTOR` 자동 매칭
- 자동 매핑 후 수동으로 수정/보완할 수 있습니다

### 6단계: 저장

**매핑 저장** 버튼을 클릭하여 설정을 저장합니다.

## 파싱 룰 편집

매핑 화면에서 **파싱 룰 편집** 버튼을 클릭하면 모달이 열립니다.

### VRL 동기화

- **VRL 동기화** 버튼을 클릭하면 aggregator.toml의 VRL 코드에서 `.data.*` 필드를 자동 추출합니다
- 추출된 필드가 드롭다운 목록에 반영됩니다
- 설비 유형별로 사용 가능한 필드가 달라집니다

### 수동 필드 추가

- 필드명을 입력하고 **추가** 버튼을 클릭합니다
- `data.` 접두사가 자동으로 붙습니다
- 불필요한 필드는 삭제할 수 있습니다
