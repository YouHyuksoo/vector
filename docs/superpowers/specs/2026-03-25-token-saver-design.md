# Token Saver Skill Design

## 목적
모든 대화에서 토큰 소모를 최소화하는 개인 skill.

## 적용 범위
- 위치: `~/.claude/skills/token-saver/SKILL.md`
- 트리거: 모든 대화에서 기본 적용

## 규칙 카테고리

### 1. 응답 간결화
- 작업 완료 후 요약/반복 금지
- 코드 변경 설명은 한 줄로 제한
- 인사/전환어/빈 문장 제거
- 도구 결과에 이미 보이는 정보 반복 금지

### 2. 파일 읽기 최적화
- 100줄 이하만 전체 읽기 허용, 그 외 limit/offset 필수
- 같은 대화 내 동일 파일 재읽기 금지
- Grep으로 위치 파악 후 Read

### 3. 도구 사용 최적화
- 단순 검색에 Agent 사용 금지
- 독립적 도구 호출은 병렬 실행 필수
- 도구 호출 수 최소화

### 4. 탐색 중복 방지
- 이전 결과에서 얻은 정보 재탐색 금지
- Agent 결과 신뢰 및 재활용

### 5. 컨텍스트 관리
- 메모리 활용하여 불필요한 탐색 회피
- gitStatus 제공 시 git log/status 반복 금지

## 설계 결정
- 규칙 목록형 채택 (체크리스트형, 원칙형 대비 간결하고 모호함 없음)
- skill 파일 자체도 최소 토큰으로 작성
